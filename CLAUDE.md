# Төслийн заавар (CLAUDE.md)

## 1. Төслийн тухай
Энэхүү төсөл нь "Бие даалт 13"-ын хүрээнд хийгдэж буй "Personal Task Tracker" юм. Төсөл нь Төлөвлөлт (Part A) болон Код бичих (Part B) гэсэн хоёр үндсэн хэсгээс бүрдэнэ.

## 2. Фолдерын бүтэц
- **partA/**: Төслийн төлөвлөгөө, архитектурын диаграмм, яагаад энэ технологийг сонгосон тухай баримтууд (ADR).
- **partB/**: Програмын үндсэн код.
  - `src/routes/`: Вэб сайтын замууд (endpoint).
  - `src/services/`: Програмын үндсэн ажиллагаа (бизнес логик).
  - `public/`: Хэрэглэгчид харагдах хэсэг (HTML, CSS, JS).
  - `tests/`: Код зөв ажиллаж байгааг шалгах тестүүд.
- **.claude/commands/**: AI-д өгөх тусгай тушаалууд (Slash commands).

## 3. Програмыг ажиллуулах тушаалууд (Part B дотор)
- **Сангуудыг суулгах**: `npm install`
- **Програмыг асаах**: `npm run dev` (Код өөрчлөгдөх бүрт сервер автоматаар дахин асна)
- **Тест ажиллуулах**: `npm test`

## 4. Код бичихдээ баримтлах дүрмүүд
- **Логик тусгаарлах**: Өгөгдлийн сантай харилцах бүх кодыг `src/services/taskService.js` дотор бичнэ. Route дотор шууд SQL бичиж болохгүй.
- **Алдаа шалгах**: Код бичихдээ үргэлж `try-catch` ашиглана. Алдаа гарвал хэрэглэгчид ойлгомжтой мессеж буцаана.
- **Нэршил**: Хувьсагч болон функцийн нэрийг жижиг үсгээр эхэлж (camelCase), харин Service-ийн нэрийг том үсгээр (PascalCase) эхэлж бичнэ.
- **Commit хийх**: Өөрчлөлт бүрээ `feat:`, `fix:`, `test:`, `docs:` гэж эхэлж бичнэ. AI ашигласан бол commit-ын төгсгөлд `Co-Authored-By: Claude <noreply@anthropic.com>` гэж заавал нэмнэ.

## 5. Хориглох зүйлс (No-Go)
- React, Vue гэх мэт хүнд хүрээ (framework) ашиглахгүй, зөвхөн энгийн JavaScript (Vanilla JS) ашиглана.
- Өгөгдлийн санг SQLite-аас өөр төрөл рүү дур мэдэн сольж болохгүй.
