# AI Usage Report: Personal Task Tracker Development

## 1. Introduction

Энэхүү тайланд "Personal Task Tracker" төслийг хөгжүүлэх явцад хиймэл оюун ухаан (AI)-ыг хэрхэн ашигласан, түүний давуу болон сул талууд, мөн гарсан техникийн асуудлуудыг хэрхэн шийдвэрлэсэн талаар дэлгэрэнгүй тусгав. Төслийн зорилго нь зөвхөн ажилладаг програм хийх бус, AI-тай үр дүнтэй хамтран ажиллах чадварыг эзэмшихэд оршсон.

## 2. AI Interaction Strategy

Төслийн туршид Claude 3.5 болон Cursor AI-г үндсэн туслахаар ашигласан.

-Prompt Engineering: AI-д даалгавар өгөхдөө контекстийг тодорхой болгож, "Node.js, Express, SQLite ашиглан Service-Oriented Architecture-оор код бичих" гэх мэт нарийн зааварчилгаа өгч байв.

-Iterative Development: Кодыг нэг дор бүгдийг нь биш, харин үе шаттайгаар (DB холболт -> CRUD -> Frontend -> Testing) хөгжүүлсэн.

## 3. Technical Challenges and Debugging 

AI-тай ажиллахад бүх зүйл шууд бэлэн болоогүй бөгөөд хэд хэдэн ноцтой алдааг өөрөө оношилж зассан.

-ReferenceError (addTask is not defined): AI-ийн өгсөн код заримдаа HTML болон JS-ийн холболтыг буруу тооцоолж байсан. JavaScript файл доторх функцүүд "Global Scope"-д ороогүйгээс болж HTML-ээс дуудахад алдаа зааж байсныг window.addTask = addTask; гэх мэтээр өөрийн мэдлэгээр зассан.

-Syntax Typos: Код дотор "ы" гэх мэт монгол үсэг санамсаргүй орсон байсныг Console ашиглан илрүүлж, арилгаснаар програмыг ажиллагаанд оруулсан.

-PATCH vs PUT: API-ийн статус өөрчлөх хэсэгт PATCH аргыг ашиглахдаа серверийн логиктой зөв уялдуулах шаардлага гарсан.

## 4. Evaluation of AI Contributions

-Давуу тал: Суурь кодыг маш хурдан гаргаж өгдөг, CSS загварчлалыг орчин үеийн хэв маягаар бичдэг, тест бичих ажлыг 80% хөнгөвчилдөг.

-Сул тал: Жижиг логик алдаа гаргадаг (Hallucination), төслийн файлуудын хоорондын уялдааг (ID, нэршил) заримдаа алддаг.

## 5. Learning Outcomes

Энэхүү бие даалтаар AI бол "бүх зүйлсээ хийлгэдэг зүйл" биш, харин "хүчирхэг багаж, хамтрагч" гэдгийг ойлгосон. Код бичихээс илүү кодыг уншиж, алдааг нь засах (reviewing & debugging) чадвар программист хүнд хамгийн чухал болохыг сурч авсан.

## 6. Conclusion

AI-ийн тусламжтайгаар төслийн хугацааг 3 дахин товчилж чадсан. Гэхдээ эцсийн бүтээгдэхүүний чанар, аюулгүй байдлыг зөвхөн хүний хяналт, туршлага л баталгаажуулж чадна. Цаашид AI-г илүү ухаалгаар ашиглаж, өөрийн суурь мэдлэгээ гүнзгийрүүлэх нь чухал гэж дүгнэж байна.
